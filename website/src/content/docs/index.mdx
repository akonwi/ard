    <div class="ard-section-heading">
      <h2 id="principles-title">What Ard makes explicit</h2>
      <p>Ard has dedicated syntax for optional values, recoverable errors, and mutable data access.</p>
    </div>

    <div class="ard-feature-grid">
      <article class="ard-feature">
        <div>
          <h3>Optional values</h3>
          <p>Optional values use <code>T?</code> rather than nil, pointers, or zero values. The absent case must be handled before the value can be used.</p>
        </div>
        <pre aria-label="Optional value example"><code><b>let</b> user: User? = find_user(id)
<b>let</b> name = user
{'  '}.map(<b>fn</b>(u) &#123; u.name &#125;)
{'  '}.or("guest")</code></pre>
      </article>

      <article class="ard-feature">
        <h3>Errors as values</h3>
        <p>Functions that can fail return Results (<code>Val!Err</code>). Callers can handle the error where recovery is possible or propagate it with <code>try</code>.</p>
        <pre aria-label="Explicit error example"><code><b>fn</b> divide(a: Int, b: Int) Int!Str &#123;
{'  '}<b>match</b> b == 0 &#123;
{'    '}<b>true</b> => Result::err("division by zero"),
{'    '}<b>false</b> => Result::ok(a / b),
{'  '}&#125;
&#125;</code></pre>
      </article>

      <article class="ard-feature">
        <h3>Mutable access</h3>
        <p>The <code>mut</code> keyword makes mutation explicit for mutable bindings, references, and methods that can modify their receiver in place.</p>
        <pre aria-label="Explicit mutability example"><code><b>impl</b> Board &#123;
{'  '}<b>fn mut</b> play(player: Str, pos: Int) &#123;
{'    '}self.cells.set(pos, player)
{'  '}&#125;
&#125;</code></pre>
      </article>
    </div>
